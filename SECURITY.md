# Güvenlik politikası

## Desteklenen sürümler

Proje aktif geliştirme aşamasındadır. Güvenlik düzeltmeleri yalnızca `main`
branch'inin güncel durumu için hazırlanır.

## Güvenlik açığı bildirimi

Bir güvenlik açığını herkese açık issue, discussion veya pull request ile
paylaşmayın. GitHub deposundaki **Security → Advisories → Report a
vulnerability** akışını kullanarak özel bildirim gönderin.

Özel bildirim düğmesi görünmüyorsa açığın ayrıntılarını paylaşmadan, yalnızca
proje sahibiyle özel iletişim kanalı talep eden kısa bir issue açın.

Bildirimde mümkünse şunları ekleyin:

- Etkilenen bileşen ve sürüm/commit
- Tekrarlama adımları veya minimal kanıt
- Beklenen etki ve erişim gereksinimleri
- Varsa önerilen çözüm

API anahtarları, erişim tokenları, çocuk verileri veya üretim kullanıcılarına
ait kişisel verileri rapora eklemeyin. Böyle bir veri yanlışlıkla açığa çıktıysa
önce ilgili anahtarı iptal edin.

Rapor alındıktan sonra kapsam doğrulanacak ve sonraki adımlar mümkün olan en kısa
sürede özel kanal üzerinden paylaşılacaktır.
